# 安全策略

William 厨房点餐系统是一个包含微信支付、用户订单与管理员鉴权的生产级项目，安全是首要关切。本文件说明漏洞报告方式与项目的安全边界。

## 报告漏洞

请**不要**为安全敏感问题开启公开 Issue。请通过以下任一方式私下报告：

- 在本仓库开启 GitHub Security Advisory（私有），**或**
- 邮件联系仓库所有者（见 GitHub 主页邮箱），附详细复现步骤与影响评估。

收到报告后会在数个工作日内确认。在修复发布前，请勿公开披露漏洞细节。

## 支持范围

仅以下情况视为本项目范围内的安全问题：

- 云函数鉴权缺失或越权：未登录用户可调用管理端函数、普通用户可访问他人订单、`loginAdmin` 可被绕过等。
- 支付链路漏洞：`createPayment` 金额被前端篡改、`payCallback` 签名校验缺失或可伪造、`cancelAndRefund` 可被恶意触发。
- 优惠券逻辑漏洞：`couponManager` 可被重复领取、核销后未失效、关联商品校验缺失。
- 桌码生成越权：`generateTableCode` 可被未授权调用。
- 管理后台鉴权缺陷：路由守卫可绕过、token 存储与校验缺陷。

## 不在范围

- 云环境 ID 被读取：`@cloudbase/js-sdk` 调用需暴露云环境 ID，属设计范畴。安全边界在云函数侧的鉴权与数据库权限规则，而非环境 ID 保密。
- 微信平台自身漏洞（基础库、支付 SDK、开发者工具）。
- 自行部署后未配置数据库权限规则导致的数据暴露：请按下方「部署安全清单」配置。
- 已在 `.gitignore` 中忽略但仍被强行提交的密钥文件：属误操作，请直接撤回并轮换密钥。

## 部署安全清单（自部署者必读）

部署本项目时必须完成以下配置，否则存在安全风险：

1. **数据库权限规则**：在云开发控制台为每个集合配置权限。
   - `admins`、`settings`：仅创建者可读写，禁止前端直接读。
   - `orders`：用户仅能读自己的订单（`openid == auth.openid`），写通过云函数。
   - `goods`、`categories`、`coupons`、`couponGoods`、`tableCodes`：所有人可读，仅管理员可写。
2. **云函数鉴权**：所有 `getAdminOrders / couponManager / generateTableCode / loginAdmin / resetDailySales` 等管理端函数，必须在 `index.js` 内校验调用者为 `admins` 集合中的有效管理员，**不能依赖前端隐藏入口**。
3. **支付校验**：`createPayment` 的金额必须由服务端从 `goods` 集合实时计算，**禁止信任前端传入的金额**。`payCallback` 必须校验微信签名并做幂等处理。
4. **AppSecret**：仅放在云函数环境变量或微信小程序后台，**绝不可出现在前端代码或 git 仓库**。
5. **管理员账户**：`admins` 集合存储的密码必须加盐哈希，禁止明文。

## 数据与隐私

- 小程序端会收集用户 openid（用于订单归属），不收集手机号、位置等敏感信息（除非业务必要）。
- 管理后台不存储用户支付卡信息——所有支付走微信原生流程。
- `database_export-*.json` 可能含订单与用户数据，已在 `.gitignore` 忽略，**禁止提交**。
